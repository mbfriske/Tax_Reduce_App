Data Collection

Due to the attractive market, property values have sky rocketed over the past few years in Dallas County. Consequently, this has caused an increase in property tax values which may warrant a review. We set out to create a tool that would allow Dallas property owners to determine the probability of winning an appeal by aligning the taxable value to the features of the property itself. Source data collected from the Dallas Central Appraisal District website.

Link

Data Exploration

Pulled data from DCAD and prepare it for training. We settled on a dataset that was collected from 2015 to 2019. We cleaned this data by removing duplicates, correct errors, deal with missing values, normalization, and data type conversions.

Choose Models/ Train Models

1. The Random Forest and Decision Trees model was ideal because we ultimately wanted a specific result (should appeal, or shouldn't appeal). This model performed well on large datasets and handled both our numerical and categorical data.
2. The Logistic Regression method was used for binary classification problems (problems with two class values in our features).
3. K-Nearest Neighbors was useful because several of our properties had similar features and this model helps to solve problems that have solutions that depend on identifying similar objects.
4. We used Support Vector Machine (SVM) for classification, regression, and outliers detection.

Evalute Models and Parameter Tuning

We used these metrics to "measure" the objective performance of our model. We tested the model against previously unseen data. We also hyperparameter tuned our model. We tuned our model parameters for improved performance.

Make Predictions

Use further testing data which have, until this point, been withheld from the model (and for which class labels are known), are used to test the model; a better approximation of how the model will perform in the real world.
